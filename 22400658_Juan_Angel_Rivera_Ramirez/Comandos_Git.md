# **Investigar 20 comandos de Github donde contenga el comando, descripcion y ejemplo de caso de uso.**

## 1. git init

**Descripción:** Crea un nuevo repositorio Git en una carpeta.

**Ejemplo:**

```bash
git init
```

**Caso de uso:** Iniciar el control de versiones en un proyecto nuevo.

---

## 2. git clone

**Descripción:** Copia un repositorio remoto a tu computadora.

**Ejemplo:**

```bash
git clone https://github.com/usuario/proyecto.git
```

**Caso de uso:** Descargar un proyecto de GitHub.

---

## 3. git status

**Descripción:** Muestra el estado actual de los archivos.

**Ejemplo:**

```bash
git status
```

**Caso de uso:** Ver qué archivos fueron modificados.

---

## 4. git add

**Descripción:** Agrega archivos al área de preparación.

**Ejemplo:**

```bash
git add archivo.txt
```

**Caso de uso:** Preparar cambios para guardarlos.

---

## 5. git add .

**Descripción:** Agrega todos los archivos modificados.

**Ejemplo:**

```bash
git add .
```

**Caso de uso:** Preparar todos los cambios del proyecto.

---

## 6. git commit

**Descripción:** Guarda los cambios realizados.

**Ejemplo:**

```bash
git commit -m "Corrección de errores"
```

**Caso de uso:** Registrar una nueva versión del proyecto.

---

## 7. git push

**Descripción:** Envía los cambios al repositorio remoto.

**Ejemplo:**

```bash
git push
```

**Caso de uso:** Subir cambios a GitHub.

---

## 8. git pull

**Descripción:** Descarga y actualiza los cambios del repositorio remoto.

**Ejemplo:**

```bash
git pull
```

**Caso de uso:** Obtener la versión más reciente del proyecto.

---

## 9. git fetch

**Descripción:** Descarga cambios sin aplicarlos.

**Ejemplo:**

```bash
git fetch
```

**Caso de uso:** Revisar actualizaciones antes de fusionarlas.

---

## 10. git branch

**Descripción:** Muestra o crea ramas.

**Ejemplo:**

```bash
git branch
```

**Caso de uso:** Ver las ramas existentes.

---

## 11. git branch nombre-rama

**Descripción:** Crea una nueva rama.

**Ejemplo:**

```bash
git branch desarrollo
```

**Caso de uso:** Trabajar en nuevas funciones sin afectar la rama principal.

---

## 12. git checkout

**Descripción:** Cambia de rama.

**Ejemplo:**

```bash
git checkout desarrollo
```

**Caso de uso:** Moverse a otra rama para trabajar.

---

## 13. git checkout -b

**Descripción:** Crea y cambia a una nueva rama.

**Ejemplo:**

```bash
git checkout -b pruebas
```

**Caso de uso:** Crear una rama y comenzar a usarla de inmediato.

---

## 14. git merge

**Descripción:** Une cambios de una rama a otra.

**Ejemplo:**

```bash
git merge desarrollo
```

**Caso de uso:** Integrar nuevas funciones al proyecto principal.

---

## 15. git log

**Descripción:** Muestra el historial de cambios.

**Ejemplo:**

```bash
git log
```

**Caso de uso:** Revisar commits anteriores.

---

## 16. git diff

**Descripción:** Muestra diferencias entre archivos.

**Ejemplo:**

```bash
git diff
```

**Caso de uso:** Ver qué se modificó antes de hacer commit.

---

## 17. git remote -v

**Descripción:** Muestra los repositorios remotos configurados.

**Ejemplo:**

```bash
git remote -v
```

**Caso de uso:** Verificar la conexión con GitHub.

---

## 18. git rm

**Descripción:** Elimina archivos del repositorio.

**Ejemplo:**

```bash
git rm archivo.txt
```

**Caso de uso:** Borrar archivos que ya no se necesitan.

---

## 19. git reset

**Descripción:** Deshace cambios preparados.

**Ejemplo:**

```bash
git reset archivo.txt
```

**Caso de uso:** Quitar un archivo del área de preparación.

---

## 20. git stash

**Descripción:** Guarda cambios temporalmente.

**Ejemplo:**

```bash
git stash
```

**Caso de uso:** Guardar trabajo sin hacer commit para cambiar de tarea.
